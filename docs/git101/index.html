Example of Git workflow

  1. Clone an existing repository

$ git clone username@host:/path/of/the/repository/to/clone

1.1 #### Clone an existing repository specifying the branch

git clone -b <branch> <remote_repo>

  1. Move the new files from the working directory to the staging area

$ git add .

  1. New commit (with HEAD as parent) from staging area

$ git commit -m "<message-as-present-statement>"

  1. Syncronize your local repository with the hosted repository

$ git push origin

Branching to create a new feature

  1. Create the local branch to work on the new feature development:

$ git branch <new-feature-branch>

  1. Select the "new-feature-branch" as active working branch:

$ git checkout <new-feature-branch>

  • Working on feature development:

<code the new feature>

$ git add .
$ git commit -m "Add <new-feature>"

  1. Create the new remote branch to be reviewed:

$ git push origin <new-feature-branch>

  1. Review and merge the local branch with the local master and push the updates to remote master:

$ git checkout master
$ git merge <new-feature-branch>

$ git push origin master

  1. Delete both local and remote "new-feature-branch" branches:

$ git branch -d <new-feature-branch>
$ git push origin --delete <new-feature-branch>

Extra

  • Inspect the content of the remote repository:

$ git remote show origin

  • Pull any changes that was pushed to the hosted repository:

$ git pull

  • Take a file from the staging-area and take it in the working directory (keep modifying):

$ git restore --staged <file-name>

  • List the local and remote branches:

$ git branch -v
$ git branch -vr
